# Тепловая карта (heatmap) — как устроена и как читать

Документ описывает чарт **«Heatmap: активность по зонам»** на дашборде Superset и расчёт в ClickHouse.

---

## 1. Что показывает чарт

Тепловая карта отвечает на вопрос: **где на поле (в какой трети) игрок чаще совершает действия** — у своих ворот, в середине или у чужих ворот.

| Ось чарта | Поле в данных | Значения |
|-----------|---------------|----------|
| **Строки (Y)** | `field_zone` | `own_22`, `midfield`, `opp_22` |
| **Столбцы (X)** | `player_name` | игроки выбранной команды (после фильтра) |
| **Цвет ячейки** | `SUM(events_count)` | сколько событий в этой зоне у этого игрока |

Источник: VIEW `rugby.v_player_zone_activity` (датасет Superset с тем же именем).

---

## 2. Откуда берутся данные

```
match_events (каждое событие: x_coord, team_id, player_id, event_type)
        │
        ▼
v_player_zone_activity  — GROUP BY игрок + field_zone + тип события
        │
        ▼
Superset heatmap_v2     — SUM(events_count), строки = field_zone
```

В витрине для каждой комбинации `(player_id, field_zone, event_type)` считается `events_count = count()`.

На дашборде тип события **не выведен отдельно** — Superset суммирует все `event_type` в одну ячейку (carry, pass, tackle и т.д. вместе).

---

## 3. Как считается `field_zone` (own_22 / midfield / opp_22)

Координата **`x_coord`** — доля длины поля **0…100** (в демо: `random.uniform(5, 95)`).

Ориентация матча в демо:

- **x = 0** — ворота **Стрелы** (`team_id = 1`, дома)
- **x = 100** — ворота **Ак Барса** (`team_id = 2`, гости)

Зона **не привязана к левому краю экрана**, а к **команде игрока**, совершившего событие:

| `team_id` | `own_22` (свои 22 м) | `midfield` | `opp_22` (чужие 22 м) |
|-----------|----------------------|------------|------------------------|
| 1 (Стрела) | x &lt; 33 | 33 ≤ x &lt; 67 | x ≥ 67 |
| 2 (Ак Барс) | x &gt; 67 | 33 ≤ x ≤ 67 | x &lt; 33 |

Фрагмент SQL (упрощённо):

```sql
multiIf(
    e.team_id = 1,
        multiIf(e.x_coord < 33, 'own_22', e.x_coord < 67, 'midfield', 'opp_22'),
    e.team_id = 2,
        multiIf(e.x_coord > 67, 'own_22', e.x_coord < 33, 'opp_22', 'midfield'),
    'midfield'
) AS field_zone
```

**Смысл меток:**

- **own_22** — действие в зоне **своих** ворот (защита / выход из обороны)
- **midfield** — середина поля
- **opp_22** — зона **чужих** ворот (атака / прессинг у их 22)

У гостей «свои» и «чужие» трети **перевёрнуты по x** относительно хозяев — это правильно для регби-аналитики «относительно команды», а не «лево/право на карте стадиона».

---


## 4. Как читать ячейки и цвет

1. **Тёмнее / ярче** — больше событий `SUM(events_count)` в этой зоне у игрока.
2. **Сравнение внутри игрока** — по строкам: где он чаще играет (оборона vs атака по полю).
3. **Сравнение между игроками** — по столбцам при одной строке `field_zone` (осторожно: у разных позиций разная природа событий).
4. **Нормализация** — в настройках чарта `normalize_across: heatmap` (по умолчанию в проекте): оттенки могут масштабироваться внутри матрицы чарта; для абсолютных чисел смотрите tooltip или таблицу в SQL Lab.

Пример проверки в ClickHouse:

```sql
SELECT
    team_name,
    field_zone,
    sum(events_count) AS events
FROM rugby.v_player_zone_activity
GROUP BY team_name, field_zone
ORDER BY team_name, field_zone;
```
